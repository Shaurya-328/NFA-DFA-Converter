Automata Theory

Foundations of Finite Automata and Formal Languages

What is Automata Theory?

Automata theory studies abstract machines and the computational problems they can solve. The most fundamental machines are Finite Automata, used to recognize patterns and regular languages. They form the theoretical foundation for compilers, lexical analysers, pattern matching, and network protocols.

Deterministic Finite Automaton (DFA)

A DFA is a finite state machine where for every state and input symbol there is exactly one transition to a next state. The machine deterministically decides whether to accept or reject a string.

Formal Definition

  • Q — Finite set of states
  • Σ (Sigma) — Finite input alphabet
  • δ (Delta) — Transition function: Q × Σ → Q
  • q₀ — Start state (q₀ ∈ Q)
  • F — Set of accepting (final) states (F ⊆ Q)
DFA = ( Q, Σ, δ, q₀, F ) 5-tuple formal definition

A DFA accepts a string if, starting from q₀, processing every input symbol leads to a state in F. Otherwise it rejects.

Non-Deterministic Finite Automaton (NFA)

An NFA is a finite automaton where multiple transitions may exist for the same state and input symbol, and ε (epsilon) transitions — transitions on empty string — may also be present. NFAs are equally powerful to DFAs but often simpler to design.

Formal Definition

  • Q — Finite set of states
  • Σ — Finite input alphabet
  • δ — Transition function: Q × (Σ ∪ {ε}) → 2^Q
  • q₀ — Start state
  • F — Set of accepting states
NFA = ( Q, Σ, δ, q₀, F ) Same structure — different δ behaviour

An NFA accepts a string if at least one possible computation path leads to an accepting state.

NFA → DFA Conversion (Subset Construction)

Every NFA can be converted to an equivalent DFA using the Subset Construction (Powerset) Algorithm. Each state of the resulting DFA represents a set of NFA states.

Algorithm Steps

  1. Start with the ε-closure of the NFA's initial state — this becomes the DFA start state.
  2. For each DFA state (a set of NFA states) and each input symbol, compute the set of reachable NFA states.
  3. Take the ε-closure of that reachable set — this forms a new DFA state.
  4. Add this new state to the DFA if it hasn't been seen before.
  5. Repeat steps 2–4 until no new DFA states are generated.
  6. Any DFA state containing at least one NFA accepting state becomes a DFA accepting state.

The resulting DFA may have up to 2n states (where n is the number of NFA states), though in practice far fewer states usually appear.
